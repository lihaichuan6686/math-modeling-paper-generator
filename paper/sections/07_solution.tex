\section{Solution Process}

The solution is generated by \texttt{src/demo\_v1.py}. The script:

\begin{enumerate}
  \item defines synthetic historical demand for three representative materials;
  \item computes four-step rolling forecasts and backtest errors;
  \item generates a rolling production plan under scenarios $k_1$ and $k_2$;
  \item saves forecast, production, service-level, and scenario-comparison tables;
  \item draws forecast, service-level, and scenario-comparison figures;
  \item writes a run summary under \texttt{runs/v1-demo/}.
\end{enumerate}

All generated artifact paths are stable so that the paper and review files can reference them directly. This is the demo-scale implementation of a production-route E paper: representative materials, forecast evidence, rolling decision logic, and scenario comparison are all generated by one reproducible entry point.
