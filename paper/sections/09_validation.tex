\section{Validation and Sensitivity Analysis}

The minimum ending inventory in the generated plan is 15 units, so the safety inventory constraint is satisfied in all eight weeks. This is a feasibility validation rather than a full robustness analysis.

The main sensitivity risk is the fixed weighting rule in the supplier-evaluation score and the fixed transport loss rates. If the reliability weight or loss-rate weight changes, supplier ordering may change. If transport loss increases, the model must order more raw material to receive the same delivered amount. A real CUMCM paper should test multiple weighting schemes and compare the resulting cost, inventory, and supplier composition.

Table~\ref{tab:demo-v1-sensitivity} perturbs all loss rates by a common multiplier. The safety inventory remains feasible in each case because the ordering rule compensates for loss, but total cost changes with the amount that must be ordered.

For a production-quality paper, the sensitivity section should be expanded in three directions. First, perturb supplier weights to test ranking stability. Second, perturb demand to test whether the safety threshold is enough under peak demand. Third, compare the greedy baseline against an optimization solver. The current v1.0 version records these needs explicitly so that the review agent can flag them instead of silently treating the demo as a final solution.
