## Converted Rules

### Paper Surface Rules

- The written paper is the only visible judging artifact.
- The abstract is a competition surface, not a ceremonial summary.
- Main numerical results and conclusions should appear in the body, not only in appendix.
- Tables and figures should be designed for comparison and interpretation.
- The abstract has initial-screening weight; keep it dense, detailed, and within the one-page ceiling.
- Overall paper quality is judged through assumptions, modeling creativity, result correctness, and clarity of expression together.

### Modeling Route Rules

- Start from the problem's required answer form.
- Prefer simple and practical routes before complex routes.
- Do not force the problem into a familiar reference template.
- Use advanced algorithms only when they improve evidence or explanation.
- Keep one central route idea across the whole paper.

### Community And Network Rules

- Public discussion can reveal traps, rough ranges, and alternate interpretations.
- External numbers are sanity signals, not target answers.
- Do not panic when others report different results.
- Ideas can be exchanged; solution text and full procedures should not be copied.

### Writing Feel Rules

- Problem restatement should be short.
- Problem analysis should explain route, not repeat background.
- Assumptions should be reasonable and used later.
- Model evaluation should highlight strengths while not hiding concrete weaknesses.
- Rejected routes can be mentioned only when they clarify the final route.
- Each section needs a visible duty: restatement clarifies outputs, analysis explains route, assumptions enable solvability, solution connects formulas to results, error analysis answers doubts, and appendix supports reproducibility.
- A 20-30 page paper should feel long because evidence, figures, tables, and interpretation are distributed across sections, not because background is repeated.
- A roughly 30-page contest solution should include abstract, keywords, restatement, analysis, assumptions, symbols, model establishment/solution, evaluation, promotion, references, appendix, and supporting files only when each item carries a real role.
- Abstract generation should explicitly cover route family, modeling idea, algorithm idea, model features or checks, and main results for all problem questions.

## Literature And Citation Discipline

Additional converted durable rules:

- Literature search should mine method ideas, variables, assumptions, validation styles, and source trails rather than hunt exact answers.
- A relevant source can be followed depth-first through its method and references when it clarifies the route.
- Citations should support method, data, domain, validation, or material software claims; decorative bibliography padding should be rejected.
- Literature search and online consensus are different layers: literature supports method/citation discipline, while public community signals support interpretation, traps, and sanity checks.
