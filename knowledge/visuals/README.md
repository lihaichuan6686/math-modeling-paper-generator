## Navigation Rule

Use `v1-6-nature-style-figure-rules.md` when the draft needs a high-quality early concept, mechanism, or model-route figure.

Use `v1-6-visual-generation-decision.md` when deciding whether a figure should be code-generated, code-native schematic, or externally polished by an image model.

Use the v1.5 visual fingerprint and recent award-paper visual rhythm notes when judging whether the rendered PDF has the right competition-paper feel.

Use the visual evidence-chain rules when a figure exists but does not support a claim, subquestion, or decision.

## Status

This folder starts as the v1.6 visual-quality layer. It should grow by converting user-side PDF feedback and strong visual examples into reusable figure specifications.
