# CUMCM

Purpose: give the generator one stable entry point for the contest-specific knowledge base.

## Read This Layer In Order

1. `cumcm-atlas.md`
2. `paper-generation-playbook.md`
3. `20-30-page-paper-blueprint.md`
4. `problem-type-paper-archetypes.md`
5. `official-paper-paradigms.md`
6. `official-style-vs-modern-draft-risk.md`
7. `next-iteration-plan.md`
8. relevant deep-reading and comparison notes under `cumcm/`

## What This Layer Does

This layer answers five questions:

1. What route family does the problem belong to?
2. What paper shape does that route need?
3. What positive official-paper pattern should the generator imitate?
4. What fake-completion risks should it reject?
5. What should be tested next?

## Navigation Rule

Use the atlas for route selection, the blueprint for structure, the paradigms note for positive templates, and the risk notes for review gates.

The deep-reading notes are supporting evidence. They should be read when a route family needs more detail, not as the first and only entry point.

## Status

This file is the doorway for the CUMCM corpus. It should stay short and stable.
