## What The Comparisons Teach

### 1. Route backbone is more important than prose style

The strongest same-problem notes show that the stable part is the route:

```text
signal -> route family -> model chain -> visible artifact -> validation
```

What changes from one paper to another is usually the rhetorical lead, not the underlying modeling closure.

### 2. Early object figures matter

The comparisons keep showing the same thing: papers feel more credible when the first strong figure identifies the modeled object, network, coordinate system, or data structure.

### 3. One problem family, one closure pattern

Evaluation papers should not be judged by whether they use TOPSIS, AHP, or entropy weighting alone. They should be judged by whether those scores feed a plan, and whether the plan is validated.

### 4. Risk notes are useful as anti-patterns

The 2024 notes are not positive templates. They are gates that help reject:

- placeholder residue;
- empty or decorative appendices;
- unsupported long PDFs;
- code screenshots without reproducible outputs;
- object-free method stacking.

## Reading Rule

When preparing a generation run:

1. read the route index;
2. read the paper-family matrix;
3. read the relevant same-problem comparison note;
4. read the official paradigms note;
5. read the relevant risk note;
6. then write the paper skeleton.

## Maintenance Rule

If a new comparison note changes how the route should be written or reviewed, add it here and promote the corresponding rule into the route index, generation loop, or completion gate.
