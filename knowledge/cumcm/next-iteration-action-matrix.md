# CUMCM Next Iteration Action Matrix

Purpose: turn the roadmap and next-iteration plan into a practical action list with evidence, target files, and the most likely follow-up decision.

This is an execution matrix, not a wish list.

## Action To Outcome Map

| Action | Primary goal | Evidence to produce | Typical follow-up decision |
|---|---|---|---|
| Strengthen rail/timetable realism | make the current Type I route more contest-real | real attachment parsing, OD assignment, capacity/tracking/dwell audits, full timetable output | keep the route, tighten the validation gate, or expand the rail demo scope |
| Add monitoring-route E demo | exercise both E-route families | monitoring-route scaffold, diagnostic figures, monitoring/policy decision tables, counterfactual evidence | add or refine the C2 route anchors and review rules |
| Expand same-problem comparisons | separate route backbone from writing voice | one new comparison note, one route or archetype update, one reusable gate or figure/table rule | update the route matrix or paper-family matrix |
| Improve figure/table/caption rules | make visuals and tables more contest-real | revised style note, caption examples, table layout rules, figure-plan refinements | tighten the figure/table family indexes |
| Polish completion gates | keep fake-completion out of the workflow | revised gate item, new finding-to-gate repair cue, better evidence mapping | update quality notes and review prompts |
| Deepen official-paper coverage | add stronger positive templates | more first-pass notes, route anchors, and style comparisons | update the official-paper index and paradigms note |
| Add niche method cards | cover recurring but missing method families | one or more detailed cards, route-family notes, and output/validation examples | update method-family and route-method matrices |

## Priority Order

Use this order when choosing what to do next:

```text
route realism
-> both-E coverage
-> same-problem comparisons
-> figure/table polish
-> completion gates
-> official coverage
-> niche method cards
```

## Evidence Rule

Every new action should end in one of:

- a route bridge;
- a method bridge;
- a section bridge;
- a page-density bridge;
- a quality bridge;
- or a demo/review artifact.

If it does not change one of those, it is probably not the next best action.
