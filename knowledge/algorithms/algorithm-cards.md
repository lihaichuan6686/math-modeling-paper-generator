# Algorithm Cards

这些卡片从本地算法资料、MATLAB 代码大全和国赛优秀论文中抽象而来。目标是让智能体能做算法选择、论文表达和质量审查，而不是简单堆算法名。

## TOPSIS 综合评价

适用题型：

- 多指标评价
- 方案排序
- 指标权重已知或可由熵权法/AHP/PCA 得到

输入：

- 方案-指标矩阵
- 指标方向：正向、负向、区间型
- 权重向量

输出：

- 每个方案到正理想解和负理想解的距离
- 相对贴近度
- 排序结果

论文表达要点：

- 先说明指标标准化方法。
- 再说明正负理想解构造。
- 给出距离公式和贴近度公式。
- 表格展示权重、贴近度和排名。

质量检查：

- 指标方向是否处理正确。
- 标准化后是否量纲一致。
- 权重来源是否解释清楚。
- 排名结论是否过度解释。

## K-means / K-means++ 聚类

适用题型：

- 商品分类
- 客户/地区/样本分群
- 数据探索和后续分组建模

输入：

- 样本特征矩阵
- 聚类数 `k`
- 距离度量

输出：

- 每个样本所属类别
- 类中心
- 类内差异和类间差异

论文表达要点：

- 说明选择特征及标准化方式。
- 说明 `k` 的确定依据，如肘部法、轮廓系数或业务约束。
- 图示聚类结果或二维降维投影。
- 解释每一类的实际含义。

质量检查：

- 是否对量纲敏感特征做了标准化。
- `k` 是否有依据。
- 聚类结果是否被用于后续模型，而不是孤立展示。

## 遗传算法

适用题型：

- 高维非凸优化
- 离散/连续混合决策
- 目标函数可计算但难以解析求解

输入：

- 决策变量编码
- 适应度函数
- 约束处理方式
- 种群规模、交叉概率、变异概率、迭代次数

输出：

- 最优个体
- 最优适应度
- 收敛曲线

论文表达要点：

- 先定义决策变量和变量范围。
- 再定义适应度函数，说明约束如何罚函数化或修复。
- 给出选择、交叉、变异和精英保留策略。
- 给出参数设置表和收敛曲线。

质量检查：

- 是否设置随机种子或进行多次重复实验。
- 是否只给一次运行结果。
- 是否比较过初始网格搜索、局部优化或其他基线。
- 是否存在参数随意设定但没有解释的问题。

## 模拟退火

适用题型：

- 组合优化
- 定价/补货/路径/排程
- 易陷入局部最优的问题

输入：

- 初始解
- 邻域生成规则
- 目标函数
- 初温、降温系数、终止条件

输出：

- 最优解
- 目标函数值
- 迭代过程曲线

论文表达要点：

- 说明为什么局部搜索容易陷入局部最优。
- 解释接受劣解的概率机制。
- 给出温度参数和邻域设计。
- 展示收敛曲线和最优方案表。

质量检查：

- 邻域定义是否符合问题变量。
- 约束是否在邻域生成时保持。
- 是否多次运行检查稳定性。

## 灰色预测 GM(1,1)

适用题型：

- 小样本趋势预测
- 数据长度较短但趋势明显
- 作为基线预测模型

输入：

- 非负时间序列
- 累加生成序列

输出：

- 预测序列
- 相对误差
- 后验差比或小误差概率

论文表达要点：

- 说明原始序列和累加生成。
- 给出白化微分方程。
- 估计参数并还原预测值。
- 用误差指标说明预测可靠性。

质量检查：

- 原始序列是否适合 GM(1,1)。
- 是否只因“常见”而使用。
- 是否与时间序列、回归或机器学习方法做对比。

## 层次分析法 AHP

适用题型：

- 主观权重确定
- 多准则决策
- 专家判断可解释的场景

输入：

- 层次结构
- 判断矩阵

输出：

- 权重
- 一致性检验结果

论文表达要点：

- 建立目标层、准则层、方案层。
- 给出判断矩阵来源。
- 计算最大特征值和一致性比例。
- 只在一致性通过后使用权重。

质量检查：

- 判断矩阵是否凭空编造。
- 一致性是否通过。
- 是否和客观赋权方法混合验证。

## 蒙特卡洛模拟

适用题型：

- 不确定性传播
- 随机过程仿真
- 概率估计
- 高维复杂情形近似验证

输入：

- 随机变量分布
- 仿真次数
- 评价函数

输出：

- 指标分布
- 均值、方差、置信区间
- 极端情形概率

论文表达要点：

- 说明随机性来源和分布假设。
- 给出仿真流程。
- 展示分布图或置信区间。
- 用于验证或辅助优化，而不是替代模型解释。

质量检查：

- 分布假设是否合理。
- 仿真次数是否足够。
- 是否设置随机种子。
- 是否把随机结果写成确定结论。

## 0-1 规划 / 整数规划

适用题型：

- 选择问题
- 采样点布设
- 排程、路径、分配
- 背包模型

输入：

- 二元或整数决策变量
- 目标函数
- 容量、预算、覆盖、逻辑约束

输出：

- 最优选择方案
- 目标函数值
- 约束满足情况

论文表达要点：

- 清晰定义二元变量含义。
- 目标函数要和题目要求直接对应。
- 约束逐条解释现实含义。
- 给出求解器或搜索策略。

质量检查：

- 变量规模是否可求。
- 约束是否遗漏互斥、容量或逻辑关系。
- 是否验证解的可行性。
