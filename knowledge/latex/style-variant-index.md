# LaTeX Style Variant Index

Purpose: turn the old style-plan idea into a usable index of paper style variants, local template traits, and writing risks.

This is a style bridge, not a formatting manual.

## Style Variants

| Style variant | What it favors | Main source of evidence | Typical risk |
|---|---|---|---|
| Official concise closure | tight question map, compact abstract, visible result loops | official-paper paradigms, official first-pass notes | under-explaining the model body |
| Local awarded-paper style | denser derivation, explicit theorem/algorithm exposition, longer appendices | `model1-case-study.md` and local template notes | copying structure without updating evidence |
| Route-heavy analytical style | strong problem analysis and route explanation before formulas | route index, archetype matrices, full-stack map | repeating route analysis too long |
| Evidence-dense research style | more figures, tables, and validation per subquestion | page density matrix, section-family index, artifact ledger | page growth from evidence is harder to control |

## How To Use

Use this sequence when deciding style:

```text
route family
-> archetype
-> page density
-> section family
-> style variant
-> review risk
```

## Local Template Cues

The local `model1` case study suggests:

- high density of method-result coupling;
- explicit section hierarchy;
- appendix-oriented reproducibility support;
- frequent direct explanation of core formulas and algorithm logic.

The official-paper notes suggest:

- concise but complete closure;
- stable question numbering;
- visible artifact per subquestion;
- appendix as support, not padding.

## Style Selection Rule

Prefer the least ornate style that still satisfies:

- the problem route;
- the validation burden;
- the page-density target;
- the artifact traceability requirement.
