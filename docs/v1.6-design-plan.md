# v1.6 Design Plan

Purpose: turn the v1.5 complete-paper loop into a layout-stable, visually credible, self-review-driven draft loop.

v1.6 is not a method-complexity release. It focuses on page rhythm, award-feel soft rules, layout control, paper-ready visuals, table safety, and final review discipline.

The soft-rule anchors are `knowledge/latex/v1-6-section-rhythm-soft-metrics.md` and `knowledge/latex/v1-6-award-feel-soft-rules.md`. They cover section-level density, per-subquestion loops, contest-style title naming, structured one-page abstract rhythm, section duties, human-team prose, defensible model depth, and appendix/code expectations.

## Target Outcome

The generated paper should look like a refined CUMCM team draft rather than a long AI technical report.

Target ranges:

| Item | v1.6 target | Hard response |
|---|---:|---|
| Full PDF length | 26-32 pages | Warn below 24 or above 32; fail below 20 or above 35 unless review justifies |
| Abstract page text | 900-1150 extracted chars | Fail below 900; warn above 1200; fail if abstract spills beyond page one |
| Body before appendix | 22-27 pages | Warn if appendix inflates length |
| References | 8-15 items when sources support it | Warn outside range |
| Appendix | 2-5 pages | Warn if raw dump or layout filler |

## v1.6 Priorities

1. Page rhythm over raw length.
2. Award-feel title, abstract, and section duties before drafting.
3. Per-subquestion model-result-validation loops before surface polishing.
4. Nature-style mechanism figures over default flowcharts.
5. Table safety over dense appendices.
6. PDF visual inspection over compile-success-only handoff.
7. First-draft density over skeleton-then-padding.

## Failure Modes From v1.5 Test

| Failure | Why it matters | v1.6 repair |
|---|---|---|
| 38-page output | Too long and partially inflated by layout | Section page-rhythm targets and PDF density gate |
| Symbol section wasted space | Weak page economy | Symbol/table compaction rules |
| Concept figure looked automatic | Weak award-paper visual fingerprint | Nature-style figure protocol |
| Plot text showed unreadable boxes | Direct visual quality failure | Figure readability gate |
| Appendix script table overflowed | Obvious layout defect | Table width gate and safe table snippets |
| Abstract slightly short | Page-one signal not fully polished | Abstract 900-1150 char target |
| Title and prose felt AI-generated | First-look credibility failure | Award-feel checklist and structured abstract rules |

## v1.6 Workflow

```text
problem intake
-> run scaffold
-> section rhythm budget
-> method and artifact plan
-> nature-style concept figure spec
-> code/results/figures/tables
-> LaTeX draft
-> compile
-> v1.6 layout check
-> visual self-review
-> repair until no fail
```

## Stop Rule

Do not hand off a paper only because `main.pdf` exists. Handoff requires:

- no LaTeX fatal errors;
- no table overflow hard failures;
- no figure readability hard failures;
- page rhythm within target or justified;
- abstract page within target;
- `reviews/layout-v16-check.md` read and reflected in `reviews/review-current.md`.
