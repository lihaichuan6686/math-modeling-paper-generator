# v1.6 Test Handoff

Purpose: shortest practical handoff for testing the v1.6 layout-and-visual-gated workflow in Claude Code.

## Start

From the cloned repository root:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\setup.ps1
powershell -ExecutionPolicy Bypass -File .\scripts\start-current.ps1 -Name current -Force
```

Then ask Claude Code:

```text
Use .codex/skills/cumcm-paper-generator/SKILL.md.
Use problems/problem.md and prompts/16_launch_v1_6.md.
Produce a v1.6 layout-and-visual-gated CUMCM-style draft.
Do not hand off only because the PDF compiled.
Run both the v1.5 PDF/source gate and the v1.6 layout gate before final review.
```

## Must Exist Before Handoff

- `runs/current/section-rhythm-budget.md`
- `runs/current/figure-style-spec.md`
- `runs/current/title-candidates.md`
- `runs/current/method-depth-plan.md`
- `runs/current/artifact-ledger.md`
- `paper/main.tex`
- `paper/main.pdf`
- `reviews/pdf-v15-check.md`
- `reviews/layout-v16-check.md`
- `reviews/review-current.md`

## Required Checks

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\check-v1.6-final.ps1 -Pdf .\paper\main.pdf -Run .\runs\current -Tex .\paper\main.tex -Log .\paper\main.log -Review .\reviews\review-current.md
```

Any `FAIL` item blocks handoff.

## What The User Should Inspect

1. Page count: target 26-32 pages.
2. Abstract: one full page, problem-by-problem, 900-1150 extracted chars.
3. Concept figure: early, readable, grouped, not generic workflow.
4. Tables: no clipping or right-edge overflow.
5. Figures: no unreadable boxes, default English titles, or tiny labels.
6. Appendix: supports traceability without inflating length.
