# v1.5 Release Checklist

Purpose: provide one concrete checklist before handing v1.5 to the user for Claude Code testing or publishing it to the private repository.

## Repository Checks

Run from the repository root:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\check-v1.5-static.ps1
```

Pass only if:

- the command exits successfully;
- no temporary smoke run remains under `runs/`;
- no temporary smoke review remains under `reviews/`;
- `git status --short` shows only intended v1.5 project changes.

## Required v1.5 Entry Files

Confirm these files exist:

- `prompts/15_launch_v1_5.md`;
- `docs/v1.5-paper-template-contract.md`;
- `docs/v1.5-method-route-contract.md`;
- `docs/v1.5-test-handoff.md`;
- `docs/v1.5-readiness-report.md`;
- `docs/v1.5-release-checklist.md`;
- `docs/v1.5-user-test-feedback-template.md`;

- `knowledge/quality/v1-5-hard-gates.md`;

- `scripts/check-v1.5-static.ps1`;
- `scripts/check-v1.5-pdf.ps1`;
- `scripts/check-v1.5-pdf.py`.

## Scaffold Check

Create a temporary run:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\new-run.ps1 -Name codex-v15-release-smoke -Force
```

Verify the run contains:

- `runs/codex-v15-release-smoke/title-candidates.md`;
- `runs/codex-v15-release-smoke/method-depth-plan.md`;
- `runs/codex-v15-release-smoke/artifact-ledger.md`;
- `reviews/review-codex-v15-release-smoke.md`;
- a review section named `v1.5 Hard Gate Verdict`;
- a review section named `Method Route Verdict`.

Clean up after the check:

```powershell
Remove-Item -Recurse -Force -LiteralPath .\runs\codex-v15-release-smoke
Remove-Item -Force -LiteralPath .\reviews\review-codex-v15-release-smoke.md
```

## Generated-Paper Check

When a paper exists, run:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\check-v1.5-pdf.ps1 -Pdf .\paper\main.pdf -Run .\runs\current -Tex .\paper\main.tex -Review .\reviews\review-current.md
```

Do not hand off if:

- `reviews/pdf-v15-check.md` has any FAIL row;
- `reviews/review-current.md` contains Unknown verdict rows;
- `paper/main.tex` contains unreplaced skeleton placeholders;
- `runs/current/title-candidates.md` has fewer than five filled title candidates;
- `runs/current/method-depth-plan.md` has no concrete result or validation artifact.

## Human Visual Check

Open or render the final PDF and check:

- title sounds like a CUMCM paper, not a task label;
- abstract is paragraph-structured and visually dense;
- important abstract results are bolded;
- early concept/mechanism/model-flow figure appears;
- figures have readable labels and captions;
- body prose does not look like a checklist;
- each central subquestion has result interpretation and validation;
- appendix contains code or a script index.

## Release Notes

Before publishing or asking the user to test, state:

- v1.5 is a static test candidate, not a proven award-level system;
- v1.5 blocks the major v1.4/v1.5 visible AI-report failures;
- the next quality jump must come from user-side generated-paper feedback recorded with `docs/v1.5-user-test-feedback-template.md` and classified with `docs/v1.5-feedback-triage-matrix.md`.

## Stop Rule

Do not keep polishing v1.5 indefinitely.

Once the static checklist passes and the entry files are wired, hand it to the user for one real Claude Code test. Use the test output to decide the next repair.
