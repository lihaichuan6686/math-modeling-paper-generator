# Review Case Examples

Purpose: show what a realistic paper-quality review should look like when a v1.2 draft still has serious weaknesses.

These are not full papers and not generic checklists. They are review-output examples that teach Claude how to:

- classify failures;
- cite evidence;
- prioritize the right fixes;
- avoid vague praise or vague criticism.

## Included Cases

1. `evaluation-to-planning-demo/`
2. `rail-timetable-demo/`
