# Methodology Checklist

Run: example-forecast-to-decision

- [ ] modeled object is a time-evolving system rather than a static table
- [ ] decision object is named explicitly
- [ ] forecast route is compared against at least one nearby alternative
- [ ] forecast output is translated into downstream decision parameters
- [ ] uncertainty is propagated into the recommendation rather than discussed abstractly
- [ ] abstract closes forecast, decision, and uncertainty
- [ ] conclusion stays within the tested horizon and scenario range
