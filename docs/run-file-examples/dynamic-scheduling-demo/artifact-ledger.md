# Artifact Ledger

Run: example-dynamic-scheduling

| Artifact | Role | Must appear in body | Supports claim |
|---|---|---|---|
| workflow figure | explain the baseline-to-update route | yes | the revised task is structurally different from the baseline task |
| baseline schedule artifact | report the normal-condition plan | yes | the baseline is feasible and comparable |
| revised schedule artifact | report the disturbed-condition plan | yes | the paper answers the update task |
| comparison table | quantify before/after changes | yes | the revision has operational meaning |
| feasibility or stress artifact | validate the revised plan | yes | the revision remains credible under the stated disturbance |
| recommendation summary table | close the decision layer | yes | the final operating advice is explicit |
