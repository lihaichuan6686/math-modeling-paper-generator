# Verification Plan

Run: example-dynamic-scheduling

## Main Checks

1. verify that both baseline and revised schedules exist as explicit artifacts;
2. verify that the disturbance trigger is stated before the update logic is used;
3. verify that before/after comparison includes at least one operational metric and one feasibility view;
4. verify that abstract and conclusion say what was preserved or improved, not only that the revision exists.

## Required Artifacts

- baseline schedule artifact
- revised schedule artifact
- comparison table or plot
- feasibility or stress artifact

## Review Warnings

- fail if the paper solves only the baseline problem
- fail if the revised schedule is presented without operational comparison
- warn if stress language appears without quantified scenario scope
