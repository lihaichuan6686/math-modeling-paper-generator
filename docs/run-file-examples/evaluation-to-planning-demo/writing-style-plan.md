# Writing Style Plan

## Style Target

- target version: `v1.2`
- route family: `evaluation to planning`
- selected style variant: `official concise closure + evidence-dense research style`
- positive sample: `2021C-comparison-C066-C085-C169-C283.md`
- contrast sample: `official-style-vs-modern-draft-risk.md`

## Core Writing Choices

- abstract density target: `one dense page with score -> plan -> audit closure`
- paragraph-driven vs bullet-driven balance: `paragraph-driven in body; lists moved to tables`
- tone target: `calm, decision-oriented, operational`
- how results will be interpreted in prose: `every result artifact should state what decision it supports`
- how limitations will be stated: `name scenario breadth and weight uncertainty directly`
- family-specific abstract skeleton: `evaluation route -> planning result -> feasibility/scenario check -> recommendation`
- preferred subquestion close-out pattern: `ranking meaning -> plan implication -> audit confirmation -> next-step input`

## Section Notes

| Section | Intended feel | What must not happen |
| --- | --- | --- |
| Abstract | closes score, plan, and validation | ending at ranking |
| Problem analysis | explains why ranking alone is insufficient | generic route slogans |
| Model establishment | score layer and decision layer are linked explicitly | disconnected models |
| Results | each artifact advances the executable recommendation | table dumping |
| Validation | feasibility and scenario language are both visible | token robustness sentence |
| Conclusion | concise, numeric, actionable | soft qualitative closing only |

## Human-Team Soft Rules

- the planning layer should feel heavier than the evaluation layer;
- the first major result should make the planning input concrete;
- each subquestion should end by saying what enters the next layer;
- recommendation language should sound operational rather than ceremonial.

## Thinness Risks

- sections likely to become too short: `validation`
- sections likely to become too bullet-heavy: `problem analysis`
- sections likely to overuse generic prose: `strengths and limitations`
- subquestions most likely to end without a real close-out sentence: `candidate-screening subsection`
