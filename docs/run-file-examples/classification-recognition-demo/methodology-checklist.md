# Methodology Checklist

Run: example-classification-recognition

- [ ] modeled object is a labeled sample system rather than a score-only table
- [ ] decision object is the recognition rule under a reported error tradeoff
- [ ] preprocessing and split policy are explicit enough to judge leakage risk
- [ ] classifier choice is justified by multi-metric comparison
- [ ] class-level or confusion-level interpretation is planned
- [ ] abstract closes preprocessing, classifier, and class-level validation
- [ ] conclusion stays within the tested sample scope
