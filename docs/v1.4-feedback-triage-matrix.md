# v1.4 Feedback Triage Matrix

Purpose: turn user-side v1.4 test feedback into the next concrete repository repair.

Use this after filling `docs/v1.4-user-test-feedback-template.md`.

## How To Use

1. Find the most visible failure from the generated paper.
2. Match it to the closest row below.
3. Inspect the listed evidence files.
4. Repair the listed rule, template, prompt, or run artifact.
5. Run `scripts/check-v1.4-static.ps1`.
6. Re-test with a fresh or repaired Claude Code run.

## Repair Priority

When several failures appear, repair in this order:

1. answer visibility and result consistency;
2. missing body result artifacts;
3. method depth and validation;
4. literature/citation support and online-consensus safety;
5. abstract and conclusion closure;
6. page rhythm and section density;
7. prose polish.

## When To Create v1.5 Work

Create a v1.5 target only when:

- the same failure appears in more than one fresh test;
- the failure comes from a general method, not one specific problem;
- the repair requires a new rule or template, not just a one-run fix.

Do not create a new version for one-off local paper cleanup.
