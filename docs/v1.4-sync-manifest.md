## Conceptual Changes

1. v1.4 becomes the top-level behavior layer.
2. v1.3 remains the methodology scaffold below v1.4.
3. Online consensus checking is inserted after initial model planning and before final paper writing.
4. Community soft rules are treated as static knowledge, not as answer sources.
5. The quality review now includes a human-feel and contest-circle gate.
6. New run scaffolds include `online-consensus-notes.md`.
7. Section-duty and visual-evidence-chain rules make section structure and figures/tables auditable.
8. Online consensus must either change concrete run files or record why public signals were rejected.
9. Abstract and closeout rules require final answers to agree across abstract, body, conclusion, and artifact ledger.
10. User-side feedback is triaged into specific repository repair targets before creating v1.5 work.
11. Local contest-workflow notes are converted into a three-role execution model: modeling, programming, and writing.
12. Remaining local `.doc` and `.ppt` experience files have a maintained extraction queue and safe absorption protocol.
13. Old-format extraction attempts are tracked separately so partial or blocked probes do not become false absorption claims.
14. Old binary Office text probes can be repeated with `scripts/probe-binary-office-text.py`.
15. Literature search and citation discipline are separated from online consensus so sources support methods, data, domain claims, and validation rather than copied answers.
16. New run scaffolds include `literature-notes.md` so source support is recorded separately from online consensus.
