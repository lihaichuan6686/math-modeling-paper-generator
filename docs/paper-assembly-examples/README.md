# Paper Assembly Examples

Purpose: bridge the gap between isolated section examples and a full v1.2 paper draft.

These notes show how a believable CUMCM-style paper is assembled from:

- run files under `runs/current/`;
- generated figures and tables;
- section-level LaTeX drafts under `paper/sections/`;
- the final abstract and conclusion closure.

They are not full papers. They are assembly maps that show how the whole draft should hold together.

## Included Families

1. `evaluation-to-planning-demo/`
2. `rail-timetable-demo/`

Each example folder contains:

- `paper-assembly.md`

## What Each Assembly Map Shows

For one route family, the map records:

- the stable question map;
- the intended section order and page rhythm;
- which run-file artifact feeds which paper section;
- which figures and tables should appear early, middle, and late;
- where each major subquestion closes;
- which claims belong in the abstract and conclusion.
