# v1.5 Test Handoff

Purpose: give the shortest practical handoff for testing whether v1.5 looks less like an AI report and more like a serious CUMCM team draft.

## Before The Test

From the repository root:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\setup.ps1
powershell -ExecutionPolicy Bypass -File .\scripts\check-v1.5-static.ps1
```

The static check must pass before treating the repository as a v1.5 test candidate.

For the fuller readiness view, read:

- `docs/v1.5-readiness-report.md`;
- `docs/v1.5-release-checklist.md`;
- `knowledge/latex/v1-5-award-paper-visual-fingerprint.md`;
- `docs/v1.5-user-test-feedback-template.md`;
- `docs/v1.5-feedback-triage-matrix.md`.

Prepare:

- one fresh problem statement;
- all attachments and data files;
- optional excellent-paper comparison sample.

## Prompt To Claude Code

Use this message:

```text
Use the repository skill at .codex/skills/cumcm-paper-generator/SKILL.md.
Read START_HERE.md and CLAUDE.md.
Use prompts/15_launch_v1_5.md as the main instruction.

Generate a v1.5 award-paper-feel CUMCM-style mathematical modeling paper draft for the active problem.

Requirements:
- read only the v1.5 Level 0 files first;
- use paper/templates/cumcm_v15_main.tex as the paper skeleton;
- create runs/current/title-candidates.md before writing paper/main.tex;
- fill method-depth-plan.md according to docs/v1.5-method-route-contract.md;
- create a contest-style title, not a direct task label;
- write a paragraph-structured abstract with 针对问题一/二/三 wording and bold key results;
- include an early concept/mechanism/model-flow figure;
- make all figures paper-ready, readable, Chinese-captioned, and traced in artifact-ledger.md;
- include Data Traps in data-inventory.md;
- include appendix code or a script index;
- compile paper/main.pdf when local LaTeX is available;
- run scripts/check-v1.5-pdf.ps1 after paper/main.pdf exists and use reviews/pdf-v15-check.md as evidence; pass -Run .\runs\current, -Tex .\paper\main.tex, and -Review .\reviews\review-current.md so title, method-depth, abstract-bold, placeholder, and review-verdict checks run too;
- write reviews/review-current.md with a v1.5 Hard Gate Verdict table;
- do not hand off if any hard gate fails.
```

## Fast Failure Signals

Mark the run as not v1.5-ready if:

- title sounds like the problem statement;
- `runs/current/title-candidates.md` is missing;
- `title-candidates.md` still contains `Unknown`, `fill`, `TBD`, or an empty selected title;
- abstract is one block or leaves page one half blank;
- LaTeX abstract has fewer than three `\textbf{...}` key result highlights;
- `paper/main.tex` still contains skeleton placeholders such as `<说明...>` or `<关键...>`;
- no concept figure appears;
- prompt words such as `问题信号识别` or `路线选择理由` appear in the paper;
- figures have tiny labels or default English plot titles;
- a central subquestion has no validation artifact in method-depth-plan.md;
- method-depth-plan.md still contains placeholder rows such as Unknown/fill/TBD;
- a recommended result depends on tiny sample groups or impossible fitted values;
- no appendix code or script index appears;
- `reviews/pdf-v15-check.md` has any FAIL item;
- review verdict tables still contain `Unknown`;
- review does not include `v1.5 Hard Gate Verdict`.
- rendered pages 1-6 do not match the visual rhythm in `knowledge/latex/v1-5-award-paper-visual-fingerprint.md`.

## What v1.5 Is Allowed To Be

v1.5 does not need to guarantee an award-level method.

It must, however, stop obvious AI-report failures:

- weak title;
- block abstract;
- unreadable figures;
- missing appendix code;
- shallow result sanity;
- template-looking body prose;
- compiled PDF treated as enough.

## After The Test

Record the user's visible-paper feedback in `docs/v1.5-user-test-feedback-template.md`.
Then use `docs/v1.5-feedback-triage-matrix.md` to decide the next smallest repair before starting another version.
