# v1.5 Paper Template Contract

Purpose: stop the running model from rebuilding the whole paper structure from scratch.

v1.5 papers should start from a neutral CUMCM paper skeleton and fill it, not improvise a new structure.

## Required Skeleton

Use or mirror:

```text
paper/templates/cumcm_v15_main.tex
```

If the final paper is written to `paper/main.tex`, copy the skeleton first and then replace placeholders.

Before replacing the title placeholder, create:

```text
runs/current/title-candidates.md
```

The file should record five candidate titles, rejected AI-like or task-label titles, and the selected contest-style title.

## Required Sections

The final paper should contain these sections unless the problem clearly does not support one:

1. title;
2. paragraph-structured abstract;
3. keywords;
4. problem restatement;
5. problem analysis with early concept figure;
6. assumptions;
7. symbols;
8. data processing and data traps;
9. model establishment and solution by question;
10. result analysis and validation;
11. sensitivity/error/robustness analysis;
12. model evaluation and improvement;
13. conclusion;
14. references;
15. appendix with key code or script index.

## Mandatory Early Concept Figure

The problem analysis section must include a concept figure, model pipeline, risk-window diagram, or variable-relation schematic.

Allowed creation methods:

- TikZ;
- matplotlib/networkx;
- graphviz;
- SVG generated by script.

Do not use AI-generated bitmap images as numeric evidence.

## Abstract Contract

The abstract must use paragraph blocks:

```latex
\begin{abstract}
<overall paragraph>

针对问题一，...

针对问题二，...

针对问题三，...

针对问题四，...

综上，...

\keywords{...}
\end{abstract}
```

Key results must use `\textbf{...}`.

## Figure/Table Contract

Every body-critical figure/table needs:

- a script/source entry in `artifact-ledger.md`;
- a Chinese caption;
- nearby interpretation;
- a subquestion link;
- a sanity note when the result has obvious risk.

## Appendix Contract

Appendix should include:

- core code snippets or a script index;
- mapping from script to figure/table;
- reproducibility notes;
- no raw code dump that replaces body explanation.

## PDF Review Contract

Before handoff, inspect the compiled PDF for:

- page-one abstract density;
- early concept figure presence;
- blank-space pages;
- unreadable figures;
- prompt-language leakage;
- appendix code presence.

Run the executable visible-quality check when a PDF exists:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\check-v1.5-pdf.ps1 -Pdf .\paper\main.pdf -Run .\runs\current -Tex .\paper\main.tex -Review .\reviews\review-current.md
```

The check writes `reviews/pdf-v15-check.md`. It inspects the PDF, run files, LaTeX source, and review verdicts when paths are provided. Include its failures and warnings in the final review.

Record results in the review under `v1.5 Hard Gate Verdict`.
