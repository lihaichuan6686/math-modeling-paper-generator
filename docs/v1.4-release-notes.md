## Workflow Changes

The v1.4 workflow is:

```text
problem
-> problem profile
-> route decision
-> model plan
-> Playwright MCP public research
-> online consensus reflection
-> figure/table and section planning
-> code and artifacts
-> LaTeX writing
-> PDF build when available
-> v1.4 quality review
-> user-side feedback
```

Playwright MCP public research is inserted after the initial model plan and before online consensus review/final writing. Public signals must be converted into run-file changes or explicitly rejected. The workflow must not use cookies, login bypass, hidden APIs, internal endpoints, private groups, paid content, or app-only content.

## Paper-Feel Changes

v1.4 expects:

- dense abstract with method-result closure for each major subquestion;
- page 2 quickly entering restatement or analysis;
- early model evidence around pages 4-6 when the problem supports it;
- artifact/prose alternation around pages 8-12;
- body-visible final answer tables and figures;
- validation or comparison artifacts when available;
- references that support method, data, and domain claims;
- appendix code that supports reproducibility rather than replacing body explanation.

- common mojibake risks in key files.

## Known Boundaries

v1.4 does not prove:

- award-level method quality;
- perfect coverage of all problem families;
- guaranteed public-source availability;
- guaranteed PDF compilation on every local LaTeX setup;
- suitability for active contest submission.

The version is intended for learning, post-contest review, authorized research, and paper-quality checking.
