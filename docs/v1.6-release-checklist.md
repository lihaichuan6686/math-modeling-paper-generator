# v1.6 Release Checklist

Purpose: decide whether the v1.6 layer is ready for user-side testing or a repository sync.

## Repository Checks

Run:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\check-v1.5-static.ps1
powershell -ExecutionPolicy Bypass -File .\scripts\check-v1.6-static.ps1
```

Both must pass.

## Generated-Paper Check

When a generated PDF exists, run:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\check-v1.6-final.ps1 -Pdf .\paper\main.pdf -Run .\runs\current -Tex .\paper\main.tex -Log .\paper\main.log -Review .\reviews\review-current.md
```

The outputs `reviews/final-v16-check.md`, `reviews/pdf-v15-check.md`, and `reviews/layout-v16-check.md` must be read before handoff.

## Stop Rule

Do not mark v1.6 ready if:

- v1.6 static check fails;
- generated run scaffold lacks `section-rhythm-budget.md` or `figure-style-spec.md`;
- the review lacks `v1.6 Layout Gate Verdict`;
- the final check report is missing or not PASS;
- the layout check has unresolved `FAIL` items;
- the paper is only longer or prettier but still has weak evidence and unsafe claims.
