```text

```

Purpose: make every result in the paper traceable to data, code, equations, and rendered output.

## Figures

| ID | Role | Source type | Paper label | Figure file | Generated by | Input data/source/prompt | Paper section | Caption draft | Status | Review notes |
|---|---|---|---|---|---|---|---|---|---|---|
| F01 | evidence/explanatory/validation/route/format/review | reproducible code/schematic/review screenshot | | | | | | | planned | |

Checks:

- Figure file exists.
- Figure has a caption.
- Figure is cited in the text.
- Axes, labels, units, and legends are readable.
- Evidence figures are reproducible from data/code.
- Explanatory figures have a recorded drawing source, diagram source, or generation prompt.
- AI-generated schematic figures are not used as fake data or fake experimental evidence.

Figure types:

- `evidence`: data plots, simulation results, optimization curves, prediction curves, confusion matrices, error maps.
- `validation`: residuals, sensitivity checks, uncertainty intervals, baseline comparisons, feasibility audits.
- `explanatory`: model workflow, geometry sketch, mechanism diagram, route map, variable relationship diagram.
- `format`: cover-independent logos or template assets, only when needed for layout testing.
- `review`: rendered PDF screenshots or layout-comparison images used only for quality inspection.

Allowed statuses:

- planned
- generated
- inserted
- rendered
- needs revision
- accepted

## Completion Rule

A run is not complete until:

1. All abstract and conclusion claims are listed in Key Results.
2. All figures and tables are listed.
3. All main code runs are listed.
4. The PDF build status is recorded.
5. Review status is either `Pass` or `Needs revision with explicit open items`.
