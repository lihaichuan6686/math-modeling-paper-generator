# v1.2 Quickstart

Use this page when you want the shortest path to a real v1.2 trial.

## Fastest Start

1. Put the problem statement in `problems/problem.md`, or pass a file to the starter script.
2. Run:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\start-v1.2-current.ps1 -Name current -Force
```

3. In Claude Code, use:

```text
Use the repository skill at .codex/skills/cumcm-paper-generator/SKILL.md.
Read START_HERE.md, CLAUDE.md, and docs/v1.2-quickstart.md.
Then use problems/problem.md to produce a v1.2 mathematical modeling paper draft and review.
Follow prompts/08_launch_v1_2.md.
```

If the first draft still reads too thin or too machine-like after review, immediately follow with:

```text
Run prompts/09_revision_v1_2.md on the active run and repair the paper before finalizing.
```

## What v1.2 Adds

A v1.2 run is not only a normal scaffold. It also requires:

- `runs/current/method-depth-plan.md`
- `runs/current/section-budget.md`
- `runs/current/writing-style-plan.md`

The machine gate will fail if these stay as untouched placeholders.

## Good First Test

When you want a quick smoke test before a real paper run:

- `scripts/run-v1-demo.ps1` for the production-route E demo
- `scripts/run-rail-demo.ps1` for the Type I rail/timetable demo
