# v1.6 Paper Template Contract

Purpose: make v1.6 papers start from a neutral, fillable, layout-safe CUMCM skeleton instead of rewriting a demo paper from scratch.

## Required Skeleton

For v1.6, copy or mirror:

```text
paper/templates/cumcm_v16_main.tex
```

Use the v1.5 skeleton only for explicit legacy v1.5 runs:

```text
paper/templates/cumcm_v15_main.tex
```

## What v1.6 Fixes

The v1.6 skeleton fixes v1.5 test pain points:

- no bare `tabular{lll}` for symbol or script-index tables;
- structured abstract with per-question paragraphs;
- required early concept/mechanism figure with a compile-safe fallback;
- explicit data-trap section;
- route-to-paper fields from `runs/current/route-decision.md`;
- validation and sensitivity section before model evaluation;
- appendix script index and core code snippet section.

## Before Drafting

Complete these run files before replacing TODO blocks:

- `runs/current/title-candidates.md`
- `runs/current/route-decision.md`
- `runs/current/award-feel-checklist.md`
- `runs/current/section-rhythm-budget.md`
- `runs/current/figure-style-spec.md`
- `runs/current/method-depth-plan.md`

The selected route should provide:

- paper spine;
- first-12-page signal;
- abstract claim shape;
- validation close;
- first-look failure to avoid.

## Replacement Rules

- Replace every visible TODO block before final handoff.
- Keep the abstract in the structured per-question rhythm.
- Replace the concept-figure fallback with a real figure when the problem supports one.
- Keep symbol and appendix tables in `tabularx` or another width-safe environment.
- Do not allow appendix code to replace body explanation.
- Do not copy prompt language such as `Paper spine` or `First-12-page signal` into the final PDF.

## Review Rules

Before final handoff:

1. Run the v1.5 PDF/source gate.
2. Run the v1.6 layout gate.
3. Inspect page one, early concept figure, wide tables, and appendix pages from the rendered PDF.
4. Confirm `reviews/review-current.md` has `v1.6 Layout Gate Verdict` with all 10 v1.6 gates marked Pass or Fail.
5. Confirm the final source has no visible TODO, Unknown, planning-field names, concept-figure fallback text, or placeholder bibliography items.
6. Confirm every `\input`, `\include`, and `\includegraphics` path resolves to an existing source or figure file.
7. Confirm `runs/current/artifact-ledger.md` lists every body-critical figure/table/key result and has no remaining template placeholders.
8. Confirm final `\title{...}` matches `Selected title:` and the abstract has per-question paragraphs, bold key results, and object/method/result keywords.
