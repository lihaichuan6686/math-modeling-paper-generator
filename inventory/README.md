# Inventory

Purpose: keep a lightweight record of source material categories, what has already been absorbed, and what still needs reading or verification.

This folder is not for copying papers into the repo. It is for tracking coverage.

## Source Buckets

Local source material currently falls into these broad buckets:

1. CUMCM historical problems and official excellent papers.
2. Local awarded-paper examples and LaTeX templates.
3. Algorithm and MATLAB reference material.
4. CUMCM experience notes and model libraries.
5. Generated run artifacts and review evidence.

See also: `source-map.md` for the current coverage ledger and `source-absorption-matrix.md` for the next conversion targets.

## Current Coverage

Already absorbed into structured notes:

- route families for evaluation/ranking, planning, forecasting, classification, geometry, online optimization, and E-type subroutes;
- official-paper first-pass notes for 2021 to 2023;

## Still Worth Tracking

Keep an eye on:

- route coverage gaps that still lack a strong official anchor;
- additional same-problem comparisons;
- any local source folder that has been read but not yet converted into a note;
- demo output folders that should be retained as evidence versus cleaned as temporary artifacts.

## Update Rule

When a new source folder, paper batch, or important route is absorbed, record it in the relevant knowledge note first, then add a short note here only if it changes coverage or reading priority.

Use `source-absorption-matrix.md` when you want the shortest path from a raw source bucket to the next structured note.

## Status

This file is the inventory doorway. Keep it short and update it only when coverage changes.
